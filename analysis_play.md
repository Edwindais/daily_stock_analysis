# A 股分析 Playbook

基于以下三部分做出的结论：

- 仓库代码：`src/analyzer.py`、`src/core/pipeline.py`、`src/search_service.py`、`src/stock_analyzer.py`、`data_provider/base.py`、`data_provider/fundamental_adapter.py`
- 运行日志：`run_log/stock_analysis_20260318.log`、`run_log/stock_analysis_debug_20260318.log`
- 你当前已有版本的 `analysis_play.md`

这份文档的目标不是“再写一版更漂亮的 prompt”，而是把现在这个项目在 A 股场景下真正会错在哪里、先修什么最值、哪些数据该补、Prompt 怎么改，拆成可执行的清单。

## 1. 先给结论

当前这套系统已经不是“只有技术面 + 简单新闻”了。它实际上已经具备：

- 技术趋势打分
- 实时行情
- 新闻检索
- 基本面聚合 `fundamental_context`
- 资金流 / 龙虎榜 / 板块榜单能力

但问题是：

1. **很多已经算出来的数据，没有真正进入 prompt**
2. **新闻检索噪音高于信息增量**
3. **Prompt 仍然是“普通趋势股思维”，对 A 股题材、ST、情绪周期、公告驱动不够敏感**
4. **当前默认流程更适合“保守趋势过滤”，不适合“识别 A 股高弹性机会”**

所以你现在的瓶颈，不是模型不够强，而是：

- 输入样本质量不够稳定
- A 股标签体系不完整
- Prompt 没有按“股票类型”分流
- 搜索结果没有做足够严格的“同股校验”和“页面类型去噪”

## 2. 这次实跑里，真实暴露了什么问题

下面这些不是推测，是日志里已经发生的。

### 2.1 新闻窗口对 A 股太短

这次运行中，搜索窗口是：

- `NEWS_STRATEGY_PROFILE=short`
- `NEWS_MAX_AGE_DAYS=3`
- 实际生效窗口 `effective_window=3`

日志里 7 只股票总共发生了 40 次新闻过滤，累计：

- `drop_old_total = 96`
- `drop_unknown_total = 96`

也就是说，很多内容不是“没搜到”，而是“搜到了但被你当前规则扔掉了”。

更具体一点：

- Bocha 的 `latest_news` 最终保留下来的条数分别是 `1,1,3,1,1,2,2`
- 对 A 股个股来说，这个上下文通常偏薄

对 A 股而言，3 天窗口适合“盘中/超短”，不适合作为日终综合判断的默认值。周末、题材切换、公告滞后披露时，3 天经常把真正重要的催化和风险过滤掉。

### 2.2 Tavily 在你当前链路里几乎没有贡献

这次日志里，Tavily 的典型情况是：

- `返回 6 条`
- `kept=0`
- `drop_unknown=6`

本次共有 **16 次** 类似情况。

这说明在你当前的 A 股中文搜索场景里：

- Tavily 不是“没搜到”
- 而是“搜到了，但日期元信息不足，被过滤规则全部清空”

所以你现有版本里“新闻几乎为空”这个判断，方向是对的，但原因要修正：

- 不是搜索服务整体失效
- 是 **Tavily + 严格日期过滤** 这一组合，在中文 A 股资讯上收益很低

### 2.3 搜索结果噪音很重，而且会误导模型

这个比“新闻太少”更严重。

在 `*ST大立(002214)` 的风险排查里，日志明确混入了：

- `*ST天微` 的监管处罚新闻

这说明当前搜索链路存在两个问题：

1. **同名前缀污染**
2. **页面类型污染**

另外，Bocha 返回了很多：

- 东方财富行情页
- 搜狐行情页
- 行业报告站点

这些内容并不是“个股事件新闻”，但会被塞进：

- `latest_news`
- `risk_check`
- `industry`

然后交给模型总结。模型会尽量“讲得像有依据”，但依据本身已经偏了。

### 2.4 当前运行里筹码分析是关闭的

这次实跑日志里明确写了：

- `筹码分布分析已禁用`

要注意两点：

1. 这次运行的**实际状态**确实是禁用
2. 但**代码默认值**不是禁用，`src/config.py` 里默认是 `true`

也就是说：

如果你主要做 A 股短中线，筹码没进模型，上层 Prompt 又在要求模型判断“筹码健康”，那模型只能靠模式猜。

### 2.5 技术分析器已经算了 MACD/RSI，但 Prompt 根本没吃到

这是当前仓库里最容易忽略、但非常实在的问题。

`src/stock_analyzer.py` 已经计算了：

- `MACD`
- `RSI`
- `MA60`

但 `src/core/pipeline.py` 的 `_enhance_context()` 只把这些字段中的一部分传给了 `trend_analysis`：

- `trend_status`

- `volume_status`
- `buy_signal`
- `signal_score`

**MACD 和 RSI 没有进入 prompt。**

这意味着：

- 你已经付出了计算成本
- 但模型决策时看不到这些信息

### 2.6 基本面能力已经接进来了，但 Prompt 只用了很小一部分

现在主流程里已经有 `fundamental_context`，包括：

- `valuation`
- `growth`
- `earnings`
- `institution`
- `capital_flow`
- `dragon_tiger`
- `boards`

但当前 `src/analyzer.py` 的 `_format_prompt()` 真正写进 Prompt 的，基本只有：

- PE / PB / 市值
- 财报摘要

没有真正展开使用的包括：

- `capital_flow`
- `dragon_tiger`

也就是说，项目已经具备了 A 股很关键的几类结构化能力，但 Prompt 还停留在“技术面 + 少量财报”的使用方式。

### 2.7 当前 Prompt 对 A 股“题材、情绪、风格”不够友好

系统 Prompt 的底层立场非常明确：

- 不追高
- MA5 > MA10 > MA20
- 回踩均线更优

这套逻辑对：

- 白马趋势股
- 中低波动趋势股
- 机构抱团中段

是有效的。

但对 A 股里真正高收益、也高噪音的几类标的，适应性不够：

- 题材龙头
- 连板票
- 首板 / 反包 / 一阳穿多线
- ST 风险博弈
- 公告驱动票
- 情绪冰点转折票

所以你现在这套系统，天然偏向：

- 少犯错
- 少追高
- 但也容易错过 A 股最有弹性的交易段

## 5. Prompt 该怎么改

### 5.1 先改思路，不要先改字数

你现在的问题不是 Prompt 太短，而是：

- 没有先做“标的分类”
- 没有先做“证据分层”
- 没有先做“新闻去噪约束”

建议把 Prompt 改成三段式。

### 第一段：先给模型做标的分类

建议让模型先判断这只票属于哪一类，再进入后续规则。

可分为：

1. `core_trend`
2. `theme_momentum`
3. `event_driven`
4. `st_risk`
5. `mean_reversion`

规则示例：

- 有 ST 标签，优先归入 `st_risk`
- 有连板 / 龙虎榜 / 主力净流入明显，优先归入 `theme_momentum`
- 有重大公告催化，优先归入 `event_driven`
- 多头排列 + 基本面稳定，归入 `core_trend`

这样做的好处是：

- 普通趋势股继续严格不追高
- 题材股不再被“一刀切”误伤
- ST 股会被单独风控

### 第二段：给不同类型不同规则

这是最关键的 Prompt 改造。

#### 对 `core_trend`

继续保留你现在的主规则：

- MA5 > MA10 > MA20
- 乖离率严格
- 回踩优先

#### 对 `theme_momentum`

不要再直接套普通股规则。

建议改为：

- 允许更高乖离率
- 必须结合：
  - 主力净流入
  - 板块热度
  - 龙虎榜
  - 涨停 / 连板状态
- 输出必须强调：
  - 高波动
  - 只能轻仓试错
  - 跌破哪个位置立刻撤

#### 对 `event_driven`

重点看：

- 公告日期
- 事件级别
- 市场是否已经反应
- 事件持续性

不能只看均线。

#### 对 `st_risk`

Prompt 里要单独加约束：

- ST / *ST 默认不按普通趋势股处理
- 除非有明确摘帽、重整、扭亏、资产注入等催化
- 否则默认高风险观察标的

这类股只靠均线会误判。

### 第三段：要求模型只使用“有效证据”

这次日志已经证明搜索噪音会污染判断，所以 Prompt 必须加：

1. 风险和利好只允许引用“当前股票自身”的信息
2. 如果新闻里主体不是该股票，不得写入 `risk_alerts`
3. 行情页、报价页、百科页，不得当作事件新闻
4. 没有日期或日期超窗，不得引用
5. 若证据不足，明确写“暂无足够事件证据”

这一段非常重要，因为它能直接压制“模型自动脑补”。

## 6. 我建议你改成什么样的 Prompt 框架

不是完整 Prompt，只给结构。

### 模块 A：标的识别

要求模型先输出：

- `stock_style`
- `is_st`
- `is_theme_momentum`
- `is_event_driven`
- `main_thesis`

### 模块 B：证据优先级

强制模型按以下顺序使用证据：

1. 公告 / 结构化事件
2. 资金流 / 龙虎榜 / 板块归属
3. 趋势与量价
4. 搜索新闻摘要

这样可以减少“新闻摘要压过硬事件”的情况。

### 模块 C：A 股专属约束

加上这些明确指令：

- 不能把普通趋势股和题材股使用同一追高阈值
- 不能把 ST 股当成普通价值股分析
- 不能用单一 PE 高低直接否定成长 / 题材票
- 如果股票属于热点主线，必须讨论“板块强度和情绪位置”
- 如果市场处于退潮期，提高买入门槛

### 模块 D：输出不只给结论，还要给“失效条件”

建议增加：

- `setup_type`
- `entry_trigger`
- `setup_invalid_if`
- `best_case`
- `base_case`
- `worst_case`

这样报告会比单纯的“买/卖/观望”更适合执行。

### 乖离率

`BIAS_THRESHOLD=5.0` 可以保留。

## 10. 最后评价你现在这套系统

它现在更像：

- 一个 **保守型趋势过滤器**

而不是：

- 一个 **真正理解 A 股风格切换和情绪周期的交易助手**

它擅长的是：

- 避开明显走坏的票
- 给普通趋势股做纪律化判断

它暂时不擅长的是：

- 发现题材主线龙头
- 区分“强趋势高风险”和“普通追高”
- 识别公告驱动与情绪反转
- 处理 ST / 事件博弈 / 连板风格

所以改进方向很明确：

- 不是推翻现有技术框架
- 而是把它从“统一规则判断所有 A 股”升级成“先识别风格，再按风格判断”
