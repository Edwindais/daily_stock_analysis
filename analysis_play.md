# A 股分析 Playbook

更新时间：2026-03-18

这份文档记录当前仓库在 A 股场景下的真实能力、已完成的第二轮改造、仍然存在的误差来源，以及下一步最值得继续做的方向。

---

## 1. 当前状态

以 `run_log_v2` 和本轮代码改造为基准，现在系统已经从“技术面 + 少量新闻”的通用模板，升级到“技术 + 结构化事件 + 资金 + 板块 + 过滤新闻”的 A 股版本。

当前可以明确确认的事实：

- `gemini/gemini-3-flash-preview` 调用链路是通的。
- GitHub workflow 现在已经显式透传 `AGENT_MODE`、`AGENT_ARCH`、`AGENT_SKILLS`、`AGENT_STRATEGY_ROUTING`、`AGENT_ORCHESTRATOR_MODE`、`LLM_TEMPERATURE`，不会再出现 Variables 设了但运行没生效的情况。
- 如果 `LLM_TEMPERATURE` 没有显式设置，而模型属于 `gemini-3-*`，系统会自动回退到 `1.0`，避免继续触发 Gemini 3 preview 的低温度警告。
- 主分析 Prompt 不再只看均线、PE 和新闻摘要，而是会联合读取公告事件、北向、融资融券、股东户数、主力资金、龙虎榜、板块强弱、MACD、RSI。

## 2. 本轮已完成

### 2.1 真值一致性

- 启动日志现在会打印：
  - 实际模型
  - 实际温度
  - 实际新闻窗口
  - 是否启用 Agent
  - Agent 架构 / 路由 / 策略列表
- 行情快照不再把“日线字段”和“实时价字段”混成一组展示：
  - 日线快照优先使用 `daily_today_snapshot`
  - 实时价格单独显示在 `price`
  - 涨跌幅优先按 `close` 与 `prev_close` 重新计算
- 筹码数据缺失时，统一渲染为“数据缺失”，不再出现 `0.0 / 0% / 中性占位`。

### 2.2 A 股结构化数据

`fundamental_context` 已新增 4 个标准块：

- `announcements`
- `northbound`
- `margin`
- `shareholder_count`

它们都统一使用：

- `status`
- `data`
- `source_chain`
- `errors`

当前已经接入的 A 股专用信息包括：

- 近 3 到 5 条公司级公告事件
  - 业绩
  - 股东变化
  - 质押
  - 监管
  - 合同订单
  - 解禁
- 沪股通 / 深股通持股变化
- 融资融券余额与方向
- 最新股东户数变化与“筹码趋集 / 趋散”判断

### 2.3 Prompt 与判断逻辑

主 Prompt 已做了以下调整：

- 股票先分类，再判断
  - `trend`
  - `sector_momentum`
  - `event_driven`
  - `turnaround_loss`
  - `st_high_risk`
- 明确规定证据优先级：
  - 结构化事件 / 公告
  - 资金与板块
  - 过滤后的新闻
  - 泛化叙述
- 明确要求：
  - 不能只因为 PE 高就否定成长 / 题材票
  - 同名不同代码资讯必须忽略
  - 公告、北向、融资、股东户数缺失时必须写“证据不足”
  - 筹码缺失时必须写“数据缺失”

### 2.4 Agent 与策略

新增 3 个 Agent 工具：

- `get_stock_event_context(stock_code)`
- `get_a_share_flow_context(stock_code)`
- `get_market_breadth(region='cn')`

并且已经让 Agent 侧优先读结构化数据，再用搜索补充。

新增策略：

- `event_driven`
- `capital_flow_resonance`
- `sector_rotation`

更新策略：

- `dragon_head`
  - 现在必须同时看板块热度、板块资金、龙虎榜 / 换手确认
- `emotion_cycle`
  - 现在会同时看市场广度、涨跌停结构、板块强弱、资金状态

策略路由新增了 `event_driven` 识别：

- 近 7 日有可验证公告事件时，会优先走事件驱动路线

### 2.5 搜索层

现在的新闻搜索做了两层额外收敛：

- 结果去重
  - URL / 标题重复不再反复灌入
- A 股来源降权
  - Tavily 在 A 股日期缺失严重时会被降权，不再重复占用多个维度

保留并继续强化的过滤项：

- 行情页 / 报价页 / 股票中心页过滤
- 同名不同代码过滤
- 泛行业报告过滤

## 3. 现在系统到底怎么判断 A 股

当前判断逻辑可以概括成三层。

### 第一层：先分股票类型

系统先判断标的是：

- 普通趋势股
- 板块动量股
- 事件驱动股
- 亏损修复股
- ST 高风险股

不同类型用不同标准：

- 趋势股更重均线、乖离率、量价
- 题材股更重板块热度、资金承接、龙虎榜
- 事件股更重公告与资金是否共振
- 亏损修复 / ST 默认更保守

### 第二层：看核心证据

系统优先看：

- 公告和公司事件
- 资金流、北向、融资、龙虎榜
- 个股所属板块、市场广度、板块强弱
- 技术面
  - MA
  - MACD
  - RSI
  - 支撑压力
  - 量比 / 换手
- 过滤后的新闻

### 第三层：给出交易动作

最终输出时会回答这些问题：

- 是买、等、减仓还是卖
- 空仓者怎么做
- 持仓者怎么做
- 理想买点 / 次优买点 / 止损位 / 目标位
- 风险来自哪里
- 当前证据够不够

## 4. 仍然存在的误差来源

虽然第二轮已经落地，但这套系统还没有到“可放心全自动判断 A 股走势”的程度。现在最大的误差还在下面几类。

### 4.1 缺少更强的盘面情绪数据

当前已经有：

- 市场广度
- 涨跌停家数
- 板块强弱

但还没有：

- 连板高度
- 炸板率
- 封单强度
- 昨日连板 / 今日晋级率

这意味着：

- 对纯情绪博弈票、龙头接力票、退潮期识别，仍然不够强

### 4.2 公告仍然是“轻结构化”，不是“深结构化”

现在已经能抓到公告事件类型，但还没有做到：

- 公告级别打分
- 利好 / 利空强弱分级
- 事件持续性判断
- 解禁规模 / 占比 / 时间点细化

所以：

- 事件驱动能力已经比之前强很多，但还不够“交易员级别”

### 4.3 资金侧还缺两类重要信息

建议下一步继续补：

- 龙虎榜席位细分
  - 是否游资主导
  - 是否机构席位参与
- 北向 / 融资的连续性判断
  - 不只是单次方向，而是连续多日趋势

### 4.4 搜索依然只是补充，不能替代公告

现在搜索噪音已经比之前低很多，但仍然只能作为补充。

原则不变：

- 公告 > 结构化资金 > 板块 > 新闻

如果后面要继续提升准确率，优先级仍然是“多加结构化数据”，而不是“继续拉长 Prompt”。

## 5. 下一步最值得做的事

按收益 / 成本比排序，建议继续做下面 5 件事。

### P1. 龙虎榜与情绪生态再深化

补：

- 机构席位 / 游资席位识别
- 连板高度
- 炸板率
- 晋级率

这是继续提升题材股 / 龙头股判断最值的一步。

### P2. 公告事件分级

补：

- 公告强弱评分
- 利好 / 利空标签
- 事件持续性
- 时间衰减

让 `event_driven` 不只是“知道有公告”，而是“知道公告值不值得交易”。

### P3. 行业相对强弱

补：

- 个股相对所属板块的超额强弱
- 板块相对大盘的超额强弱

这样能更清楚区分：

- 板块热但个股不强
- 个股强但脱离板块
- 板块和个股共振

### P4. 回测闭环

把当前输出信号沉淀成：

- 事件驱动回测
- 资金共振回测
- 板块轮动回测

没有回测闭环，A 股策略最后容易停留在“看上去合理”。

### P5. 参数按场景分层

建议不要只用一套默认值。

至少拆成三类：

- 趋势股模式
  - 更严乖离率
  - 更重均线
- 题材动量模式
  - 更重板块与资金
- 事件驱动模式
  - 更重公告与兑现节奏

---

## 6. 当前建议的默认参数

如果以“日终 A 股复盘 + 第二天计划”为目标，当前更合理的默认值是：

- `NEWS_STRATEGY_PROFILE=medium`
- `NEWS_MAX_AGE_DAYS=7`
- `AGENT_MODE=true`（如果你确实要走策略链路）
- `AGENT_STRATEGY_ROUTING=auto`
- `AGENT_ARCH=multi`（如果你能接受更长耗时）
- `ENABLE_CHIP_DISTRIBUTION=true`（接口稳定时）

温度建议：

- Gemini 3 系列：
  - 不建议无脑降到 `0.1`
  - 如果没显式配置，保持 `1.0` 更稳
- 如果要提升 JSON 稳定性：
  - 优先减少 Prompt 冲突
  - 优先减少脏上下文
  - 优先补结构化证据

---

## 7. 一句话结论

现在这套系统已经从“通用股票分析器”往“A 股可用的结构化分析器”走了一大步，但它的真正上限不在于再写更长的 Prompt，而在于继续补：

- 更强的盘面情绪数据
- 更深的公告结构化
- 更细的资金行为分层
- 回测闭环

如果继续按这个方向迭代，准确率提升空间仍然很大。
